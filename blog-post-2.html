¿Que son las cadenas como estructuras de datos en computación?

Autor: Mario Almánzar (23-0221)

Las cadenas o strings son un tipo de estructuras de datos que representan texto. Esto quiere decir que pueden ser una serie o secuencia de caracteres (letras, números y símbolos). Como este tipo de datos representa el texto, su utilización es fundamental para manipular data textual en múltiples lenguajes de programación. Dependiendo del lenguaje, las cadenas pueden ser consideradas tipos de datos o como estructuras de datos. La diferencia radica en que “estructura de datos” hace referencia a la manera en la que se organizan y se almacenan en la memoria estos datos para representar una secuencia de caracteres, mientras que “tipos de datos” se refiere a que son un tipo de dato proporcionado por el lenguaje en sí para representar y manipular texto.

Tipos de cadenas.

Las cadenas pueden ser mutables o inmutables. dependiendo siempre del lenguaje de programación en cuestión. En Python, por ejemplo, las cadenas son inmutables, por lo que una vez creada una cadena, no se puede modificar. En C, por dar un contraejemplo, esto no ocurre, ya que las cadenas con mutables y se pueden modificar en cualquier momento.

Codificación de caracteres.

La codificación de caracteres es un proceso donde se asignan números (códigos) a caracteres individuales en un conjunto de caracteres, como letras, números, símbolos y caracteres especiales. Esto es imperativo para que las computadoras entiendas las cadenas de texto, ya que estas sólo entienden y manipulan números.

Algunos tipos de codificaciones de caracteres son:

- ASCII: El American Standard Code for Information Interchange fue una de las primeras codificaciones de caracteres, ampliamente utilizadas y estándar en la computación. ASCII asigna valores del 0 al 127 a caracteres comunes del alfabeto inglés, números, signos de puntuación, y algunos caracteres especiales.

- Unicode: Este es el estándar que permite asignar un número único a cada caracter, independiente sel idioma, plataforma o programa. Esto permite representar casi todos los caracteres que son utilizados en la escritura a nivel global.

- UTF-8: El Unicode Transformation Format - 8 bits es una variante de Unicode muy conocida que utiliza de 1 a 4 bytes para representar un caracter. Rs compatible con ASCII y con los sistemas existentes.

- UTF-16 y UTF-32: Son otras variantes de Unicode donde, la UTF-16 utiliza de 2 a 4 bytes para representar un caracter y es ampliamente utilizada en Windows, mientras que la UTF-32 utiliza 4 bytes para cada caracter y representa cada caracter como un entero de 32 bits.
